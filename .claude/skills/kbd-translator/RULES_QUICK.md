## 1. ОТРИЦАНИЕ (-къым, мы-)

### Базовые правила

| Форма | Использование | Позиция |
|-------|---------------|---------|
| **-къым** | Финитные формы (индикатив, конъюнктив) | **После** TAM маркеров |
| **мы-** | Инфинитные формы (императив, оптатив, причастия) | **Перед** основой |

### Структура с -къым

```
[PRAGM]-[GEOM]-[ARGS]-[STEM]-[TAM]-КЪЫМ

Примеры:
кIуэ-р-къым   (идти-PRES-NEG)       = "не идет"
кIуэ-н-къым   (идти-FUT-NEG)        = "не пойдет"
кIуэ-а-къым   (идти-PERF-NEG)       = "не пошел"
кIуэ-ат-къым  (идти-PERF2-NEG)      = "не пошел (тогда)"
кIуэ-рэт-къым (идти-IMPERF-NEG)     = "не шел (тогда)"
кIуэ-нутэ-къым (идти-COND-NEG)      = "не пошел бы"
кIуэ-гъэну-къым (идти-PROB-NEG)     = "наверное, не пошел"
```

### Структура с мы-

```
мы-[STEM]

Примеры:
мы-кIуэ          = императив "не иди!"
мы-кIуэ-мэ       = условное "если бы не шел"
мы-щIэ-жь        = причастие "не делающий"
мы-кIуэ-у        = деепричастие "не идя"
```

### ⚠️ ДВОЙНОЕ ОТРИЦАНИЕ = УТВЕРЖДЕНИЕ!

**мы- + -къым = модальное утверждение** (обязательность, неизбежность)

```
Неправильно (калька с русского):
"Я не могу не согласиться" → *сызэгъэпсэфынмыхьэнукъым

Правильно (двойное отрицание):
мы-гъэпсэфын-хьэну-къым
NEG-соглашаться-мочь-NEG
= "Я обязательно соглашусь" / "Я должен согласиться"
```

**Типы двойного отрицания:**

1. **Модальный:** "обязательно", "неизбежно"
   - `къыхуигъэкIуэнмыхьэнукъым` = "обязательно скажешь"

2. **С отрицательными местоимениями:** усиление отрицания
   - `зыми...мыщIэкъым` = "никто не знает"

### Validation checklist

- [ ] -къым идет **после** TAM маркера?
- [ ] При двойном отрицании результат = утверждение?
- [ ] Не копирует ли корпус русскую структуру "не могу не..."?

## 2. ПОРЯДОК СЛОВ (SOV/OSV)

### Базисный порядок: SOV

```
Подлежащее + Объект + Глагол
   S           O        V

ЩIалэ-м   тхылъы-р  иджынущ
парень-ERG книга-ABS прочтет
```

### Когда порядок СТРОГО ФИКСИРОВАН

**1. Собственные имена без маркеров:**
```
Мухьэмэд Хьэсэн къеджащ  = "Мухамед Хасана позвал"
   S       O       V        (ФИКСИРОВАНО!)

❌ Хьэсэн Мухьэмэд къеджащ = "Хасан Мухамеда позвал" (другой смысл!)
```

**2. Индефинитная конструкция:**
```
Жэм лъакъуэ шкIэ иукIыркъым
коровья_нога теленок не_убивает
     S         O        V
```

**3. Многоместные глаголы:**
```
Порядок: S → O_косвенный → O_прямой → V

Нурдин Хьэсэн Анзор къыхуешэ
   S    O_косв O_прям   V
"Нурдин Анзора к Хасану приводит"
```

### Когда порядок МОЖЕТ ВАРЬИРОВАТЬСЯ

**1. С маркерами -р/-м (абсолютив/эргатив):**
```
SOV: ЩIалэ-м  тхылъы-р  иджынущ  (базовый)
OSV: Тхылъы-р ЩIалэ-м   иджынущ  (топикализация)

Маркеры четко различают роли → порядок гибкий
```

**2. Личные местоимения 1/2 лица:**
```
Сэ фэ фыслъэгъуащ  = "Я вас видел"
Фэ сэ фыслъэгъуащ  = "Вас я видел" (фокус)

Глагол кодирует роли (ф-ы-с-) → порядок свободен
```

### Validation checklist

- [ ] Есть маркеры -р/-м? Если НЕТ → порядок строго SOV
- [ ] Собственные имена без маркеров → порядок фиксирован
- [ ] Корпус не нарушает фиксированный порядок?

## 3. ЭРГАТИВНОСТЬ (-м/-р маркеры)

### Правило маркирования

| Тип глагола | Подлежащее (S) | Дополнение (O) |
|-------------|----------------|----------------|
| **Переходный** | **-м** (эргатив) | **-р** (абсолютив) |
| **Непереходный** | **-р** (абсолютив) | - |

### Примеры

**Переходный глагол:**
```
ЩIалэ-м   тхылъы-р  иджынущ
парень-ERG книга-ABS прочтет
   S         O         V

ЩIалэ-м   Iэщэ-р    елъэIащ
парень-ERG гость-ABS увидел
```

**Непереходный глагол:**
```
ЩIалэ-р   кIуащ
парень-ABS пошел
   S       V

Псэ-р     хуабэ
душа-ABS  хорошая
```

### OSV порядок (также частотен)

Особенно с глаголами: `исын` "сжечь", `тхьэлэн` "задушить", `укIын` "убить"

```
ЩIалэ-р   псы-м    итхьэлащ
парень-ABS вода-ERG задушил
   O         S         V

"Парень в воде утонул" (вода = агенс!)
```

### Validation checklist

- [ ] Переходный глагол → S в **-м**, O в **-р**?
- [ ] Непереходный глагол → S в **-р**?
- [ ] Корпус не путает маркеры?

## 4. СЛОЖНЫЕ ПРЕДЛОЖЕНИЯ

### Основной принцип

**Кабардинский предпочитает ИНФИНИТНЫЕ конструкции** вместо финитных придаточных!

### Условные предложения: -мэ

```
плъэкI-а-мэ         = "если бы смог"
кIуэ-мэ             = "если идет"
мы-кIуэ-мэ          = "если не идет"

У-иплъэн плъэкI-а-мэ, си псэ-псынэм...
2SG-видеть мочь-PAST-COND моей души-родник...
"Если бы ты смог заглянуть в родник моей души..."
```

### Временные конструкции

**-ху (когда):**
```
кIуэ-ху-м           = "когда идет"
кIуэ-ху-а-м         = "когда пошел"

Iэщ су-кI-мэ, и кIуэцIыр къызитыжырт
скот я-бью-когда его внутренности он_мне_давал
"Когда я забивал скот, он отдавал мне внутренности"
```

**нэужь (до того как):**
```
кIуэ нэужь          = "до того как пойдет"
```

**-рэм и пIалъэ (после того как):**
```
кIуэ-рэ-м и пIалъэ  = "после того как шел"
```

### Целевые конструкции: папщIэ

```
лъэIэсын папщIэ      = "чтобы достичь"
(инфинитив + послелог)

Апхуэдэ мурадхэм лъэIэсын папщIэ, нэщанэ ящI ди щIалэгъуалэр
таких целей достичь чтобы мишенью становится наша молодежь
"Чтобы достичь таких целей, мишенью становится наша молодежь"
```

### Причинные союзы

```
сыту жыпIэмэ        = "потому что"
сыт щхьэкIэ жыпIэмэ = "по какой причине"
```

### Validation checklist

- [ ] Использована инфинитная конструкция?
- [ ] Правильный суффикс/послелог (-мэ, -ху, папщIэ)?
- [ ] Корпус не копирует русскую финитную структуру?

## 5. ПОСЛЕЛОГИ

### Основные послелоги

| Русский | Кабардинский | Контекст | Пример |
|---------|--------------|----------|--------|
| в (внутри) | **щIэ** | Замкнутое пространство | унэ-м щIэ "в доме" |
| на (поверхности) | **тет** | Контакт с поверхностью | Iэстолы-м тет "на столе" |
| под | **пкъыр** | Ниже объекта | жыгы-м пкъыр "под деревом" |
| над | **IэкIи** | Выше объекта | унэ-м IэкIи "над домом" |
| с (инструмент) | **-кIэ** (INSTR) | Инструмент | дыщэ Iуданэ-кIэ "золотой ниткой" |
| с (совместность) | **IэкI** | Совместное действие | абы IэкI "с ним" |
| к (направление) | **-м** (дат.) | Направление к объекту | унэ-м "к дому" |
| чтобы (цель) | **папщIэ** | Целевая конструкция | лъэIэсын папщIэ "чтобы достичь" |

### Примеры использования

```
унэ-м щIэ               = "в доме" (внутри)
Iэстолы-м тет           = "на столе"
жыгы-м пкъыр            = "под деревом"
дыщэ Iуданэ-кIэ хидыкIащ = "золотой ниткой вышила"
абы IэкI                = "с ним" (совместно)
лъэIэсын папщIэ         = "чтобы достичь"
```

### Validation checklist

- [ ] Правильный послелог для контекста?
- [ ] Порядок: существительное + послелог (не как в русском)?
- [ ] Корпус не путает послелоги?

## 6. КАУЗАТИВНАЯ ДЕРИВАЦИЯ (гъэ-)

### Базовое правило

**Каузативный префикс гъэ- = "заставлять/делать так чтобы"**

**Позиция:** `[PRAGM]-[GEOM]-гъэ-[ARGS]-[STEM]`

### Примеры

```
Базовый → Каузатив

лъагъун    "видеть"      → гъэ-лъагъун  "показывать"
псэун      "жить"        → гъэ-псэун    "оживлять"
хуэдэн     "быть похожим" → гъэ-хуэдэн "уподоблять"
кIуэн      "идти"        → гъэ-кIуэн    "вести"
```

### Сложный пример с рефлексивом

```
зе-зы-гъэ-хъуэжыф = "изменяться"

зе-      (REFL себя)
-зы-     (REL)
-гъэ-    (CAUS)
-хъуэжыф (изменять)

= "заставлять себя изменяться" → "изменяться"
```

### Validation checklist

- [ ] гъэ- после GEOM, перед ARGS?
- [ ] Значение = каузация?
- [ ] Корпус не путает гъэ- с другими префиксами?
